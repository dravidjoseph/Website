Paradigm Name Parent Paradigm Explanation
Imperative N/A Explicitly describes actions of a program. Uses sequences of statements to modify program state.
Declarative N/A Programs describe what they want, not how to get what they want. Tend to lack side-effects.
Structured Imperative/Declarative Statements that are imperative/declarative. Get's rid of GOTOs. Uses for loops or if statements.
Procedural Imperative/Structured Based on procedures (functions) which can be called from anywhere in the code.
Functional Structured/Declarative Highly similar to procedural, but doesn't possess state so more like declarative.
Object-Oriented Imperative Data structures form objects, which contain methods and attributes. They are created/instantiated from classes, and are treated as nont structured.
Logic Declarative Engine answers questions based on facts and rules.
Scripting N/A Fast, generally designed for non-CS people to get things done quickly.
